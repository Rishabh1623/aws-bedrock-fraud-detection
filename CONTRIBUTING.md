# Contributing to Fraud Detection RFT

Thank you for your interest in contributing! This project is designed as an AWS Solutions Architect portfolio piece, but contributions are welcome.

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs
- Include detailed steps to reproduce
- Provide AWS region and service versions
- Share relevant logs (redact sensitive info)

### Suggesting Enhancements
- Open an issue with the "enhancement" label
- Describe the use case and benefits
- Consider AWS best practices and costs

### Pull Requests
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test thoroughly (include test results)
5. Update documentation
6. Commit with clear messages
7. Push to your fork
8. Open a Pull Request

## Development Guidelines

### Terraform
- Follow HashiCorp style guide
- Run `terraform fmt` before committing
- Add comments for complex resources
- Update outputs.tf for new resources
- Test in dev environment first

### Python
- Follow PEP 8 style guide
- Use type hints
- Add docstrings for functions
- Include error handling
- Write unit tests

### Documentation
- Update README.md for major changes
- Add architecture diagrams if needed
- Include cost implications
- Provide examples

## Testing Checklist

Before submitting a PR:
- [ ] Terraform plan succeeds
- [ ] No security vulnerabilities
- [ ] Cost impact documented
- [ ] Documentation updated
- [ ] Code formatted
- [ ] Tests pass

## Code of Conduct

- Be respectful and inclusive
- Focus on constructive feedback
- Help others learn AWS
- Share knowledge generously

## Questions?

Open an issue or reach out via email.

Thank you for contributing!
